# Plan: `/model` and `/effort` Bot Commands

## Goal

Add Telegram bot commands that let the owner inspect and override the OpenCode model and effort for the current topic.

The overrides should be sticky per topic, like `/plan`, and should fall back to the topic's context defaults from `config.yaml` when unset.

## Recommended Behavior

- `/model` shows the current effective model for this topic and whether it comes from a topic override, context default, or OpenCode server default.
- `/model <provider/model>` validates the provider-qualified model with `split_provider_model()` and stores it as this topic's override.
- `/model reset` removes the topic override and restores the context/default behavior.
- `/effort` shows the current effective effort for this topic and its source.
- `/effort <level>` validates the level against `EFFORT_LEVELS` and stores it as this topic's override.
- `/effort reset` removes the topic override and restores the context/default behavior.
- Already queued `TurnJob`s can keep their snapshotted model/effort; future turns should use the latest override from `Router.resolve()`.

## Implementation Steps

1. Add persistent per-topic override storage in `SessionStore`.
   - Create a separate table, for example `topic_overrides`, keyed by `(chat_id, thread_id)`.
   - Store nullable fields for `model_provider`, `model`, and `effort`.
   - Keep this separate from `topic_sessions` so overrides can be set before a session exists and survive stale-session recreation via `store.delete()`.
   - Add accessors such as `get_overrides()`, `set_model_override()`, `reset_model_override()`, `set_effort_override()`, and `reset_effort_override()`.
   - Normalize `None` thread IDs with `thread_key()` just like existing plan-mode methods.

2. Add router helpers for effective model/effort.
   - Add methods on `Router` to read, set, and clear the current topic overrides.
   - In `Router.resolve()`, merge store overrides over `ctx.provider_model` and `ctx.effort` before returning `ResolvedSession`.
   - Keep context binding behavior unchanged.

3. Add bot command handlers in `apps/backend/src/balam/bot.py`.
   - Import and reuse `EFFORT_LEVELS` and `split_provider_model` from `balam.contexts`.
   - Add `_handle_model()` and `_handle_effort()` near `_handle_status()` or `_handle_plan()`.
   - Use the current message's `chat_id` and `message_thread_id`, matching `/plan` and `/status`.
   - Reply with short usage messages on invalid input.
   - Add `model` and `effort` to `BOT_COMMANDS`.
   - Register `CommandHandler("model", _handle_model, filters=allowed)` and `CommandHandler("effort", _handle_effort, filters=allowed)` in `build_application()`.

4. Update status output.
   - Change `/status` to report effective model and effort instead of only raw context defaults.
   - Include override source only if that keeps the output readable.

## Tests

Add focused tests rather than broad end-to-end tests.

- `apps/backend/tests/test_store.py`
  - Overrides default to unset.
  - Model override round-trips and is scoped by chat/thread.
  - Effort override round-trips and is scoped by chat/thread.
  - Clearing either override is idempotent.
  - Overrides survive `store.delete()` of the session row.
  - General-topic normalization treats `None` and `GENERAL_THREAD_ID` consistently.

- `apps/backend/tests/test_router.py`
  - `resolve()` returns context defaults when no overrides exist.
  - Model override replaces only provider/model.
  - Effort override replaces only effort.
  - Clearing overrides returns to context defaults.
  - Overrides are isolated per topic and survive recreated sessions.

- `apps/backend/tests/test_bot.py`
  - Import the new private handlers for direct handler tests.
  - `/model` with no args reports the effective model.
  - `/model <provider/model>` stores the override and future router resolution uses it.
  - `/model reset` clears the override.
  - Invalid `/model` values return usage/error text and do not mutate state.
  - `/effort` with no args reports the effective effort.
  - `/effort <level>` stores the override and future router resolution uses it.
  - `/effort reset` clears the override.
  - Invalid `/effort` values return allowed levels and do not mutate state.
  - `BOT_COMMANDS` includes `model` and `effort`.
