# Tech debt inventory

_Snapshot taken 2026-07-13, `main` @ c4355c2. Backend is 9,060 lines of Python
across 28 modules; frontend is a small React/Vite app. Findings are ordered by
leverage (impact × how often the area changes), not by severity alone._

## How this was measured

- **Churn** = number of commits touching a file across all 126 commits. High
  churn + high line count = a hotspot where debt compounds fastest.
- Signals gathered: LOC/function counts, `git log` churn, duplicated
  vocabularies, broad `except`, `# type: ignore`, timing constants, type-sync
  mechanism, gitignore hygiene.

---

## Tier 1 — structural, high-leverage

### 1. `bot.py` is a god module (1,655 lines, 51 functions, **34 commits — #1 churn**)

One file holds: the message handler, **every** slash-command handler (`/context`,
`/new`, `/plan`, `/status`, `/model`, `/effort`, `/rename`, `/diff`, `/browser`,
`/cancel`, `/delete`), all inline-keyboard callback handlers (approvals,
questions, delete paging/confirm), topic naming, keyboard construction, cleanup
scheduling, and `build_application`. It is both the largest and most-changed
file, so almost every feature edit lands here and the merge/reasoning surface is
huge.

**Direction:** split by concern — e.g. `commands/` (one module per command
group), `callbacks.py` (inline-keyboard routing), `topics.py` (naming/rename/
create). `build_application` becomes a thin registrar. This is the single
highest-value refactor.

### 2. `streamer.py` is the second god module (1,336 lines, 44 functions, 23 commits — #2 churn)

Owns draft transport, live-edit fallback, tool-burst collapsing, reasoning
overflow ordering, tool label rendering, and finalization. The
"answer-stays-at-tail" invariant and collapsed-tool-stream logic are subtle and
concentrated here (both have dedicated memories). Worth extracting the
rendering/label layer from the transport/flush layer.

### 3. Tool & permission vocabulary is duplicated across 4+ files

The same conceptual tool set (`read`/`edit`/`bash`/`grep`/…) is redefined,
in slightly different string forms, in:

| File | Shape |
| --- | --- |
| `opencode_tools.py` | `Tool` enum **and** `Permission` enum |
| `permissions.py` | permission list / `parse_allowed_tool` |
| `streamer.py` | `_TOOL_DISPLAY` (tool → label) |
| `agent/claude_sdk_backend.py` | `_WIRE_TOOL` (SDK name → wire) + `_CATEGORY` |

Adding or renaming a tool means editing all of them, and the SDK↔OpenCode name
mapping (`LS`→`list`, `MultiEdit`→`edit`) lives only in `_WIRE_TOOL`. There is no
single source of truth for "what tools exist and how each backend spells them."

**Direction:** one canonical tool registry (name, wire form, display label,
permission category) that all four consumers derive from.

## Tier 2 — worth doing

### 4. Two agent backends duplicate normalization logic

`opencode_backend.py` and `claude_sdk_backend.py` each translate their runtime's
tool inputs/names into the shared `balam.agent.events` vocabulary. Some helpers
are already shared (`collapse_mcp_name`, `_normalize_input` bridges `file_path`↔
`filePath`), but display/category maps diverge. As backends are the pluggable
seam (ADR-0014), keeping the normalization contract in one place — closer to
`events.py` — would reduce drift as either runtime evolves.

### 5. Frontend type-sync is manual and can silently drift

`packages/shared/src/api.ts` is generated by the `gen:api` npm script
(`openapi-typescript` over the backend's dumped OpenAPI). It is **not** wired
into CI or a pre-commit/build step, so it only updates when someone remembers to
run it. Generated file is dated Jun 11; `server.py` has changed since. Nothing
fails loudly when they diverge.

**Direction:** run `gen:api` in `typecheck`/CI and fail if the working tree
changes (drift check), or generate on build.

### 6. `balam.db` sits untracked in the repo root and is not gitignored

`git check-ignore` reports it is **not** ignored — it shows up as `??` in every
status and is one `git add .` away from being committed. Add `balam.db` /
`*.db` to `.gitignore`.

## Tier 3 — minor / keep an eye on

- **Broad `except Exception` (~40 uses).** Audited a sample: most in `streamer.py`
  and `bot.py` **do** log at `debug` with `exc_info=True` or have an explanatory
  comment (e.g. native-draft fallback). Not silent swallowing today, but the
  count is high enough that a new bare handler could hide a real error unnoticed.
  Low priority; revisit if debugging gets hard.
- **`# type: ignore` at seams** — `server.py:227` casts `None` to a `Router` for a
  test-only construction; `config.py:153` `call-arg` for env-sourced settings.
  Both are load-bearing and commented; fine, just noted.
- **Scattered timing constants** — `draft_interval`, `asyncio.sleep(0.05)`,
  `httpx.Timeout(connect=10, …)`, `wait_for_ready(timeout=30, interval=0.5)`.
  Not centralized; harmless but would be easier to tune from one place.
- **`config.example.yaml` / `.env.example` drift** — not checked here; worth a
  periodic diff against `config.py`'s validated fields.
- **Deployed `.env` leaks into 5 tests** (found 2026-07-18, pre-existing at
  90e4251): `Config` in tests is built via pydantic-settings, which reads the
  real `.env`, so `BALAM_PUBLIC_URL` being set fails
  `test_miniapp.py` (3 tests: `*_without_public_url`, `*_localhost_text_only`,
  `*_plain_url_without_shortname`) and
  `test_agent_tools.py::test_send_file_markdown_without_public_url_sends_without_button`;
  `AGENT_BACKEND=claude_sdk` fails
  `test_config.py::test_agent_backend_defaults_to_opencode`. Fix direction: have
  the `make_config` fixture (and direct `Config()` constructions in tests) pass
  `_env_file=None` so tests are hermetic regardless of the machine's `.env`.

## What is notably clean (not debt)

- Test coverage is broad: **21 test modules**, one per source module including
  the two backends, permissions, approvals, streamer, and router.
- ADRs are thorough and referenced from code comments — the *why* is captured.
- `api.ts` (frontend) is a hand-written typed client that imports generated types
  from `@balam/shared`; it is not a parallel type definition.
- Broad excepts largely log rather than swallow (see Tier 3).

## Suggested order of attack

1. Add `balam.db` to `.gitignore` (5 min, removes an accident risk).
2. Wire `gen:api` drift check into CI (small, stops frontend/backend divergence).
3. Introduce the canonical tool registry (#3) — unblocks safe backend work.
4. Split `bot.py` by command group (#1) — the big one; do it incrementally,
   one command module at a time, leaning on the existing `test_bot.py`.
5. Extract `streamer.py`'s rendering layer (#2).
